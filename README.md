# 部署说明

这份说明面向想把工具部署到公网的人。部署后，买家可以通过二维码打开点单页；摊主通过后台处理订单。

项目不会内置你的真实网址、后台地址、收款码或订单数据。每个人部署后都会拥有自己的网址、自己的数据库和自己的后台密码。

## 开源前检查

公开仓库前，请确认不要上传这些内容：

- `data/` 文件夹
- `*.sqlite3` 或 `*.db` 数据库文件
- `.env` 环境变量文件
- 真实收款码图片
- 真实订单导出表
- 你自己的后台入口地址

本项目已经在 `.gitignore` 里忽略了这些常见数据文件，但如果你使用 GitHub 网页手动上传，仍然需要自己确认没有拖错文件。

## 本地运行

适合先在电脑上试用流程。

1. 安装 Python 3.10 或更新版本。
2. 双击 `启动摆摊工具.bat`。
3. 买家页打开：

```text
http://127.0.0.1:8765
```

4. 后台默认打开：

```text
http://127.0.0.1:8765/admin
```

5. 默认后台密码：

```text
123456
```

第一次进入后台后，请立刻修改密码。

如果手机和平板和电脑在同一个 Wi-Fi 下，可以用电脑的局域网 IP 访问，例如：

```text
http://192.168.x.x:8765
```

本地运行不需要服务器，但买家离开这个网络后就访问不到。

## 公网部署需要什么

你需要一个能运行 Python 或 Docker 的平台，例如：

- Render、Railway、Fly.io 等云平台
- 腾讯云、阿里云、华为云等云服务器
- 其他支持 Python Web Service 的平台

这个项目没有外部 Python 依赖，核心启动命令是：

```bash
python app.py
```

程序会读取平台提供的端口环境变量 `PORT`。如果平台没有提供，默认使用 `8765`。

## 重要：数据要持久保存

商品、订单、摊位设置、促销规则和后台密码都会保存在数据库里。

默认数据库位置：

```text
data/booth.sqlite3
```

如果你的云平台重启后会清空项目文件，请设置一个持久数据目录，并配置环境变量：

```text
BOOTH_DATA_DIR=/data
```

同时给 `/data` 配置持久磁盘或持久卷。

如果没有持久磁盘，服务重启或重新部署后，订单和设置可能会丢失。

## Render 参考配置

创建 Web Service 时可以参考：

```text
Language: Python 3
Branch: main
Root Directory: 留空，除非你的代码放在仓库子目录
Build Command: 可留空；如果平台要求填写，可以填 python --version
Start Command: python app.py
```

环境变量建议设置：

```text
ADMIN_PATH=/换成只有你知道的后台路径
```

如果配置了持久磁盘，建议再设置：

```text
BOOTH_DATA_DIR=/data
```

部署完成后，Render 会给你一个网站地址。买家二维码只使用首页地址，例如：

```text
https://你的网站地址
```

后台地址使用你设置的 `ADMIN_PATH`，例如：

```text
https://你的网站地址/换成只有你知道的后台路径
```

## Docker 方式

项目内置 `Dockerfile`，适合支持 Docker 的平台或云服务器。

镜像默认设置：

```text
HOST=0.0.0.0
PORT=8765
BOOTH_DATA_DIR=/data
```

请给 `/data` 挂载持久存储，并按需要设置：

```text
ADMIN_PATH=/换成只有你知道的后台路径
```

## 后台入口与密码

默认后台入口是：

```text
/admin
```

开源后，别人能从代码和文档知道默认入口。因此正式使用时强烈建议设置：

```text
ADMIN_PATH=/换成只有你知道的后台路径
```

默认后台密码是：

```text
123456
```

第一次上线后请立刻进入后台修改密码。

## 上线后的检查清单

1. 打开买家首页，确认商品页能正常加载。
2. 打开后台，确认后台路径不是默认 `/admin`。
3. 使用默认密码登录后，立刻修改后台密码。
4. 上传摊位 Logo、欢迎语、微信收款码和支付宝收款码。
5. 添加商品、库存、标签和赠品。
6. 设置摊员名单。
7. 试下一单，确认取单码、小票、订单看板和拣货流程正常。
8. 试导出订单明细和商品汇总。
9. 用手机流量打开买家页，确认不在同一 Wi-Fi 下也能访问。
10. 把买家首页网址生成二维码，贴在摊位上。

## 忘记后台密码

请查看：

```text
PASSWORD_RESET.md
```

重点是：密码保存在运行网站的数据库里，不保存在 GitHub 代码里。忘记线上密码时，需要在运行网站的服务器或云平台里执行重置脚本。
