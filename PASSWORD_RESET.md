# 重置密码说明

后台密码保存在运行网站时使用的数据库里，不保存在 GitHub 代码里。

所以忘记密码时，要在“运行这个网站的地方”重置，而不是在 GitHub 里重置。

## 本地版本

如果是在自己电脑上运行：

1. 关闭正在运行的摆摊工具。
2. 双击 `重置后台密码.bat`。
3. 后台密码会恢复为默认密码：`123456`。
4. 重新进入后台后，建议立刻在【摊位】里改成自己的密码。

也可以在项目文件夹里运行：

```bash
python reset_admin_password.py
```

## 服务器版本

如果已经部署到服务器，必须进入服务器执行：

```bash
python reset_admin_password.py
```

例如腾讯云轻量服务器，需要先登录服务器，进入这个项目所在文件夹，再运行上面的命令。

重置后后台密码会恢复为默认密码：`123456`。重新登录后，建议立刻在【摊位】里改成自己的密码。

## Render 等云平台

如果平台提供 Shell/Console，就在 Shell/Console 里进入项目目录并运行：

```bash
python reset_admin_password.py
```

如果平台没有 Shell/Console，就需要用平台支持的方式进入运行环境，或者临时重新部署并保留/处理数据库。正式办展前建议确认自己选择的平台能否方便地操作数据库文件。

## 开源给朋友时

朋友拿代码部署后，会拥有自己的网址、自己的数据库和自己的后台密码，不会和你的线上网站共用。

但如果大家都使用默认后台路径 `/admin`，别人看过代码后就知道后台入口的默认写法。因此正式部署时建议设置环境变量 `ADMIN_PATH`，改成只有自己知道的后台路径。

例如：

```text
ADMIN_PATH=/your-private-admin-path
```

设置后，后台地址就会变成：

```text
https://你的网站地址/your-private-admin-path
```

买家二维码只放买家首页地址，不要把后台地址做成二维码公开。
